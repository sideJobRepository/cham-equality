[통합 테스트 코드 작성 요청]
1. $1 에 대한 통합 테스트 시나리오 목록을 깊게 생각해서 제안해줘. 여러 계층(adapter, application, domain)이 함께 동작하는 흐름을 중심으로 작성해.
2. 제안을 사용자가 검토한 후에, 사용자에게 이대로 작성할 것인지 확인을 요청해.
3. 사용자가 승인한다면 $2에 통합 테스트코드를 작성해줘.
4. 테스트코드 작성 후 테스트 실행은 코드 확인 후 사용자가 직접 수행한다.

[통합 테스트 작성 가이드]
- @SpringBootTest를 사용하여 스프링 컨텍스트를 로드하는 통합 테스트를 작성한다.
- API 엔드포인트 테스트의 경우 MockMvc 또는 WebTestClient를 활용한다.
- 데이터베이스 연동 테스트의 경우 @Transactional을 활용하여 테스트 간 데이터 격리를 보장한다.
- 외부 서비스 의존성은 필요시 WireMock 등을 활용하여 모킹한다.
- 테스트 데이터 생성은 FixtureMonkey를 활용한다.
